# Information Architecture: Relationship Intelligence (Dashboard Zone 2) v2

**Version**: 2.0
**Input**: Product Manager Feedback (2026-01-12)
**Goal**: Restructure Zone 2 to explicitly show Membership demographics and Sales contribution per tier.

---

## 1. Visual Layout & Structure

The section is split into two primary cards (Widgets).

| Zone | Title (CN/EN) | Key Information | Chart Type |
| :--- | :--- | :--- | :--- |
| **2A (Left)** | **Membership Distribution**<br>(会员规模分布) | Total/Active Counts<br>Counts per Tier | Stacked Column Chart<br>(Total + Active overlay) |
| **2B (Right)** | **Value Contribution**<br>(消息贡献 / 价值贡献) | Sales Amount per Tier<br>Sales % per Tier | Donut Chart or Horizontal Bar |

---

## 2. Component Detailed Breakdown

### 2.1. Widget A: Membership Distribution (Left)

**Purpose**: To show the scale of the membership and the quality (activity) of each tier.

#### **Summary Header**
*   **Metric 1**: **Total Members** (总会员数)
    *   *Value*: Integer (e.g., 12,450)
*   **Metric 2**: **Active Members** (活跃会员数)
    *   *Value*: Integer (e.g., 8,200)
    *   *Context*: Active % (e.g., "65% Rate")

#### **Main Visualization: Tier Breakdown**
*   **Axis**: Tier Name (X-Axis).
*   **Data Series 1 (Background Bar)**: **Total Members per Tier** (各等级会员人数).
    *   *Tooltip*: "Bronze: 5,000 Members".
*   **Data Series 2 (Foreground Bar / Line)**: **Active Members per Tier** (各等级活跃会员数).
    *   *Tooltip*: "Bronze: 2,500 Active".
*   **Interaction**: Hovering over a bar highlights the ratio of Active vs Total for that specific tier.

---

### 2.2. Widget B: Value Contribution (Right)

**Purpose**: To demonstrate the revenue impact of each tier.

#### **Main Visualization: Contribution Chart**
*   **Type**: **Donut Chart** with Legend OR **Horizontal Bar Chart**.
*   **Data Points**:
    *   **Sales Amount** (各等级销售额): Absolute value (e.g., $50,000).
    *   **Sales Percentage** (各等级销售额占比): Share of total member sales (e.g., 40%).

#### **Breakdown Table / Legend**
List elements corresponding to the chart:
1.  **Tier Name** (Label)
2.  **Sales Amount** (Value)
3.  **Contribution %** (Percentage)

*Example Row*:
`[Gold Icon] Gold Tier  |  $125,000  |  45%`

## 3. Data Dictionary Updates

### New Data Requirements
The `metrics` object needs to be enriched to support these views.

| Field | Type | Description | Source |
| :--- | :--- | :--- | :--- |
| `tierData` | Array | List of Tier objects. | Backend / Aggregation |

#### **Tier Object Structure**
```typescript
interface TierMetric {
  id: string;
  name: string;       // e.g. "Gold"
  memberCount: number; // Total members in this tier
  activeCount: number; // Active members in this tier
  totalSales: number;  // GMV generated by this tier
  salesPercent: number;// Pre-calculated (or calculated on FE)
}
```

---

## 4. Design Guidelines

*   **Color Palette**:
    *   *Membership Bars*: Use Tier Colors (Bronze/Silver/Gold) for distinctions, or a unified Brand Blue for "Total" and Brand Green for "Active".
    *   *Sales Chart*: Use a monochromatic scale (Darkest = Highest contributing tier) or distinct tier colors.
*   **Layout**:
    *   Zone 2A should take up **60-65%** of the width (Detail heavy).
    *   Zone 2B should take up **35-40%** of the width (Summary heavy).
